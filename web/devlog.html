系统架构

本地 Python 管线每日 10:00 自动运行，数据写入 data/，推送到 GitHub，触发 Pages 部署。

launchd 10:00
  └── schedule/run_daily.sh
        ├── pull_openrouter.py    → data/rankings_openrouter.json
        ├── pull_rss.py (14源)   → data/news_*.json, community_*.json, research_*.json
        ├── pull_stocks.py        → data/stocks.json
        ├── pull_github_trending.py → data/github_trending.json
        ├── pull_huggingface.py   → data/huggingface_trending.json
        └── pull_score.py (claude -p)
              → data/digest.json      (去重+打分+摘要)
              → data/pinned.json      (高分自动置顶，≥0.88，保留14天)
  └── git push → GitHub Actions → Pages 部署

Python 环境：pipeline/.venv/bin/python（3.9，有所有依赖）。系统 python3 是 3.14，没有依赖，不要用。

问答记录（关键决策）
Q
卡片黄色和白色是什么意思？
A：relevance_score ≥ 0.8 显示黄色背景（#fefce8），表示 Claude 认为重要且多源验证；<0.8 白色普通卡片。设计重构后此区分方式将改为信源角标+分数徽章。
Q
股票图表是日线还是月线？
A：用 yfinance period="30d" 拉取约21个交易日的日线收盘价，以迷你折线图展示，横轴为近一个月走势。
Q
Ticker 里 1.00 是什么单位？
A：relevance_score 是 0.0→1.0 的相关度指数（满分1.0），由 Claude 打基础分，再按多信源加权（每多一个信源 +0.08，上限1.0）。1.00 即满分，说明该条目极度相关且多源验证。Ticker 已改为显示信源列表而非分数——所有进入 Ticker 的条目已是 ≥0.88 的高分，不需要再显示数字。
